Security

Script Verification

Always verify a script's integrity before running it on your system.

Why check the hash?

Every Afina script has a unique hash — a digital identifier of the file. If a script has been modified after publication (intentionally or as the result of an attack), its hash will not match the reference value on this page.

  • Script substitution in transit via an insecure channel (MITM attack)
  • A tampered copy hosted on a third-party mirror
  • Unauthorized modification of the file after publication
Instructions

How to verify authenticity

The process is the same on all platforms. Calculate the hash of the downloaded file and compare it with the reference value in the Scripts section below.

Verification steps

  1. Obtain the hash associated with your downloaded script.
  2. Go to the Scripts section below and find the corresponding script.
  3. Paste the hash into the verification tool below and click Verify. A match confirms the hash is authentic.
One mismatched character is enough to consider the file untrustworthy. If in doubt, re-download from the official source and verify again.
